# v0.2 Loop 09 验收：设置与诊断页

## 本轮范围

- 扩展 provider 诊断结构：
  - `state`: `ready` / `fallback` / `missing`
  - `envVars`: 需要配置的变量名
  - `docs`: 缺口文档位置
- 新增 `GET /api/settings/diagnostics`。
- 前端 PROVIDERS 区增加设置摘要：
  - ready / fallback / missing 数量
  - 缺失原因
  - 需要补的 env 变量名
- 同步更新 `docs/v0.2-missing-inputs.md` 的诊断页映射。

## 验收命令

```powershell
node --check server.js
node --check public\app.js
```

结果：通过。

```powershell
$env:PORT='18109'
$env:DEEPSEEK_API_KEY='secret_should_not_leak'
node server.js
Invoke-RestMethod 'http://127.0.0.1:18109/api/settings/diagnostics'
```

结果摘录：

```json
{
  "ready": 1,
  "fallback": 2,
  "missing": 5,
  "deepseekState": "ready",
  "leaked": false,
  "hasEnvVars": true
}
```

## 验收结论

- UI 不需要日志即可看到缺少哪些配置。
- API 只返回变量名，不返回密钥原文。
- 缺口文档和诊断项已同步。

## 注意

- `Invoke-WebRequest` 在一次 raw 内容检查时出现 PowerShell 空引用异常；已改用 `Invoke-RestMethod | ConvertTo-Json` 验证同一目标。
