# v0.2 真实功能端到端验收

## 结论

Claudio v0.2 已完成可直接运行的真实功能验收路径：

- `npm start` 可启动当前真实功能服务。
- 无 `.env` 时服务可启动，并明确显示 missing/fallback。
- 有 `LOCAL_MUSIC_DIR` 时可扫描和流式播放本地真实音频。
- 无 DeepSeek key 时明确回退到本地规则脑，不伪装 AI 成功。
- 用户输入可生成真实队列。
- 喜欢/跳过事件会写入状态，并影响后续推荐。
- 前端桌面页可打开，控制台无 error。

## 本轮修复

- `/api/health` 版本从 `0.1.0` 修正为 `0.2.0`。
- `npm start` 改为直接运行 `server.js`，避免依赖未安装或未构建时无法启动当前可用版本。
- 保留 `npm run start:compiled` 给后续 Fastify/React 完整构建版本使用。

## 验收记录

### 无 `.env`

临时移走 `.env` 后启动服务：

```json
{
  "health": true,
  "ready": 0,
  "fallback": 2,
  "missing": 6,
  "localConfigured": false,
  "localReason": "缺少 LOCAL_MUSIC_DIR"
}
```

### 本地真实音频

用临时生成的 WAV 文件验证 `LOCAL_MUSIC_DIR`：

```json
{
  "localConfigured": true,
  "localTracks": 1,
  "contentType": "audio/wav",
  "streamBytes": 16044,
  "chatProvider": "local",
  "chatQueue": 1,
  "beforeQueue": 1,
  "afterSkipQueue": 0
}
```

说明：

- `afterSkipQueue` 从 1 变 0，证明跳过事件会影响后续推荐。
- 测试期间备份并恢复了 `data/state.json`，不提交运行时状态。

### 前端页面

Browser 本地打开 `http://127.0.0.1:18110/`：

## 仍需用户后续补齐

### 依赖

按用户要求，依赖有问题不继续反复安装。本轮只执行一次：

```powershell
npm run build
```

结果：

```text
error TS2688: Cannot find type definition file for 'node'.
```

对应缺口仍见 `docs/v0.2-missing-inputs.md` 的“本地依赖安装缺口”。后续网络稳定后执行：

```powershell
npm install
```

### 外部能力

- `DEEPSEEK_API_KEY`：未提供时使用本地规则脑。
- 真实 TTS provider：未提供时使用浏览器 TTS 回退。
- ASR provider：未提供时使用浏览器语音识别能力。
- 网易云真实播放能力：当前只完成元数据接入；真实播放仍受登录、版权、VIP、地区限制影响。
- 天气、日历、UPnP：未接入时都明确显示未配置。
