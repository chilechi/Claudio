# Loop 03 验收记录

日期：2026-06-20

## 已完成

- 新增 `music/local-music-provider.js`。
- 新增 `/api/music/local/scan`。
- 新增 `/api/tracks/:id/stream`。
- `.env.example` 增加 `LOCAL_MUSIC_DIR`。
- 当前前端在配置本地音乐目录后优先使用本地扫描结果作为队列。
- 播放按钮会调用真实 `<audio>` 播放；未配置音频时明确提示需要配置 `LOCAL_MUSIC_DIR`。

## 验收命令

语法检查：

```powershell
node --check music\local-music-provider.js
node --check server.js
node --check public\app.js
```

无配置场景：

```text
GET /api/music/local/scan
```

结果：

```json
{
  "configured": false,
  "reason": "缺少 LOCAL_MUSIC_DIR",
  "tracks": []
}
```

有配置场景：

- 临时生成 `data/test-music/claudio-test.wav`
- 设置 `LOCAL_MUSIC_DIR=D:\workspace\Claudio\data\test-music`
- 调用 `/api/music/local/scan`
- 调用第一首歌的 `streamUrl`

结果：

```json
{
  "configured": true,
  "count": 1,
  "title": "claudio-test",
  "playable": true,
  "type": "audio/wav",
  "riff": "RIFF"
}
```

## 清理

临时测试音频目录 `data/test-music` 已删除，没有提交音频文件。

## 结论

Claudio 已具备第一个真实可播放音乐源：本地音乐目录。没有配置目录时不会伪造播放能力。
