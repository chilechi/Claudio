# v0.3 全链路 E2E 和切换默认入口验收

## 结论

v0.3 工程化迁移的构建、核心逻辑、React Web、mobile readiness 均已通过检查；默认入口暂不切换到新 Nest server。

原因：当前本机 `node_modules/fast-json-stringify` 包内容不完整，`package.json` 指向 `index.js`，但实际缺少该文件，导致 `npm run start:server` 无法启动。按严格验收标准，新 server 不能启动时不能把 `npm start` 切到 v0.3。

## 阻塞项

### 新 Nest server 启动失败

命令：

```powershell
$env:PORT='18110'
npm run start:server
```

结果：

```text
Cannot find module 'D:\workspace\Claudio\node_modules\fast-json-stringify\index.js'
```

处理：

- 不切换 `npm start`。
- 保留 `npm start` 指向 v0.2 fallback。
- README 已说明 v0.3 新入口和当前依赖缺口。
- 缺口继续记录在 `docs/phases/v0.2/missing-inputs.md`。

### 自动化浏览器音频播放限制

Loop 07 已确认本地音频流返回标准 Range 响应，浏览器也能识别 `duration=4`。但内置自动化浏览器点击播放时返回 `NotAllowedError`，未能自动证明 `audio.paused=false`。

处理：

- 页面不伪装播放，保持 `待命` 并显示错误类型。
- 后续需要在普通浏览器手动点击或可授予媒体播放权限的自动化环境中复测。

## 默认入口策略

当前保留：

```json
"start": "node server.js"
```

新入口保留：

```json
"start:server": "node apps/server/dist/main.js"
```

切换条件：

1. 本地依赖重装后 `npm run start:server` 可启动。
2. 新 server + React Web 全链路通过。
3. 播放器真实音频在普通浏览器中通过播放、暂停、切歌、进度、音量验收。
4. 再将 `npm start` 切到新 server。
