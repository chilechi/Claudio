## 已解决项

### 新 Nest server 启动依赖

曾经阻塞项：`node_modules/fast-json-stringify/index.js` 缺失。

处理：

- 删除损坏的 `node_modules/fast-json-stringify` 单包目录。
- 执行 `npm install fast-json-stringify@6.4.0`。
- 将 `fast-json-stringify` 写入直接依赖，避免后续安装时缺少 Fastify 运行入口。
- 复测 `npm run start:server`：通过。

### 自动化浏览器音频播放限制

Loop 07 已确认本地音频流返回标准 Range 响应，浏览器也能识别 `duration=4`。但内置自动化浏览器点击播放时返回 `NotAllowedError`，未能自动证明 `audio.paused=false`。

处理：

- 页面不伪装播放，保持 `待命` 并显示错误类型。
- 后续需要在普通浏览器手动点击或可授予媒体播放权限的自动化环境中复测。

## 默认入口策略


```json

```
